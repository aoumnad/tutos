Introduction
Arduino est une petite carte de prototypage dotée d'un microcontrôleur ATMEGA et une interface USB-Série qui permet de la connecter à un port USB du PC pour  téléverser (upload) les programmes dans l'ATMEGA. La carte possède des connecteurs permettant d'accéder facilement aux E/S du Microcontrôleur

            

Il existe plusieurs types de cartes Arduino. Cet article les présente d'une façon très complète et colorée. Je me permet de reprendre son tableau récapitulatif.


Les exemple de ce tutoriel seront testés sur le Arduino UNO dont les caractéristiques sont:

∎ Processeur:  Atmega 328 avec:
•  32ko de mémoire programme (flash)
•  2ko de mémoire de données (SRAM)
•  Horloge 16 Mhz

∎ Les Entrées Sorties:
•  13 E/S numériques dont 6 peuvent fonctionner comme sorties PWM (Pulse Width Modulation)
•  6 entrées analogiques,
•  Un bus de communication Série asynchrone (UART)
•  Un Bus de communication synchrone I2C/SPI

∎  Limitation de courant
• Le courant max recommandée sur une sortie Arduino est 20mA. En aucun cas il ne doit dépasser 40 mA.
• Le courant max que toutes les sorties réunies peuvent fournir ne doit pas dépasser 200mA
• Le courant max que toutes les sorties réunies peuvent recevoir ne doit pas dépasser 400mA

∎  Logiciel
Pour programmer l'Arduino, on dispose d'un environnement de développement intégré (IDE) constitué essentiellement de:
• Un éditeur de texte qui permet au programmeur de saisir son programme,
• Un compilateur C adapté au microcontrôleur ATMEGA. Le compilateur permet de traduire le code source écrit en C en un code  exécutable constitué d'une suite de codes machine compréhensibles par le microcontrôleur ATMEGA.
• Une fois le programme compilé, l'IDE permet de d'envoyer (téléverser) le code exécutable dans la mémoire programme de l'ATMEGA afin d'y être exécuté.

∎  Télécharger et installer l'IDE Arduino
L'IDE de programmation est gratuit, on peut le télécharger ici: https://www.arduino.cc/en/Main/Software
Il est disponible en deux versions, un installateur (.exe) ou une version zippée (.zip) qu'il suffit d'extraire dans un dossier. Je crois que la version (.exe) est plus rapide à télécharger car elle moins volumineuse que la version (.zip). Son installation ne pose pas de problème particulier.

∎  Installation du driver de la carte Arduino
Quand on branche l'Arduino à un port USB du PC, l’installation du driver se fait automatiquement. Si cette opération échoue pour une raison ou une autre,  procéder comme suit pour installer le driver :
•    Ouvrir le gestionnaire de périphérique: clic droit sur ordinateur→ propriétés → Gestionnaire de périphérique
•    Localiser l'équipement qui n'a pas de driver. Si vous avez des doutes, débrancher le câble USB, c'est celui qui disparaît. Rebranchez le câble USB, il réapparaît
•    Clic droit dessus → mettre à jour le pilote → rechercher un pilote sur mon ordinateur → Sélectionner le répertoire « drivers » dans le dossier d'installation de IDE → confirmer

Que l'installation se fasse automatiquement ou pas, il est bon d'aller dans le gestionnaire de périphériques pour noter le numéro de port COM assigné par le driver. On en aura besoin pour configurer l'IDE arduino. Noter que si on branche l'arduino sur un autre port USB, le numéro de port COM affecté par le driver peut etre différent.

Les Entrées/Sorties Numériques

Pour utiliser une E/S numérique, il faut d'abord la configurer en entrée ou en sortie à l'aide de la fonction pinMode().
Ensuite, on utilise la fonction digitalRead() pour lire l'état d'une entrée et digitalWrite() pour fixer l'état d'une sortie

  • pinMode(pin, state);
    • Permet de configurer une E/S numérique en Entrée ou en Sortie
    • pin: numéro de l'E/S  (type = Integer)
    • state:  INPUT ou 0 pour Entrée,  OUTPUT ou 1 pour Sortie
  • digitalWrite(pin, state);
    • Permet de forcer une sortie au niveau logique haut (5V) ou au niveau logique bas (0V)
    • pin: numéro de l'E/S (int)
    • state: HIGH ou 1 pour le niveau haut,  0 ou LOW pour le niveau bas
  • val = digitalRead(pin);
    • Retourne (int) l'état logique d'une entrée numérique (0=LOW ou 1=HIGH)
    • pin: numéro de l'entrée (int)

Contrôler une sortie PWM


Un signal PWM (Pulse Width Modulation) ou MLI (Modulation de largeur d'impulsion) est un signal  dont le rapport cyclique TH /T  est réglable.
Arduino UNO possède 6 sorties (3, 5, 6, 9, 10 et 11) pouvant fonctionner en PWM.


Pour générer un signal PWM, on utilise la fonction:

 analogWrite(pin, rs);

  •   pin :   numéro de la patte PWM
  •   rs :      entier compris entre 0 et 255 qui définit le rapport cyclique

Après un appel de la fonction analogWrite(), le signal PWM reste actif sur la sortie PWM.
Pour le modifier, il faut faire un autre appel à cette fonction,
Pour l'arrêter on peut appeler la fonction digitalWrite() ou digitalRead()  sur la patte considérée.
remarque: On n'est pas obligé de configurer la broche en sortie à l'aide de  pinMode()

Tester une entrée Numérique
Les capteurs numériques, des interrupteurs, des bouton poussoirs...doivent être branchés sur des entrées numériques.
On va faire un petit exemple avec deux boutons poussoirs B1 et B2 et deux LEDs L1 et L2.

Avant de commencer, il faut préciser qu'il y a deux façons de connecter un bouton poussoir ou un interrupteur pour délivrer une impulsion numérique:
  • Avec le branchement active-low, le bouton délivre un niveau haut au repos, et un niveau bas quand il est cliqué,
  •  Avec le branchement active-high, le bouton délivre un niveau bas au repos, et un niveau haut quand il est cliqué,
  • Personnellement, je préfère le branchement active-low car il n'y a jamais un lien direct entre l'entrée du microcontrôleur et le 5V

   

Le bouton B1 permet d'allumer/éteindre la led L1, le bouton B2 en fait de même avec L2


#define B1PIN 8
#define B2PIN 12
#define L1PIN 1
#define L2PIN 6
boolean L1STATE = false, L2STATE = false;

void setup() {
  pinMode(B1PIN, INPUT);   // bouton 1
  pinMode(B2PIN, INPUT);   // bouton 2
  pinMode(L1PIN, OUTPUT);   // LED 1
  pinMode(L2PIN, OUTPUT);   // LED 2
  digitalWrite(L1PIN, LOW);  // éteindre les LEDS au départ
  digitalWrite(L2PIN, LOW);
}

void loop() {
 
  if( digitalRead(B1PIN) == 0){               // si le bouton est cliqué ...
    L1STATE = !L1STATE ;                          // état actuel <-- inverse de l'état précédent
    digitalWrite(L1PIN, L1STATE);          // LED <-- état actuel
    while( digitalRead(B1PIN) == 0);      // attendre la fin du click
  }
  if( digitalRead(B2PIN) == 0){
    L2STATE = !L2STATE ;
    digitalWrite(L2PIN, L2STATE);
    while( digitalRead(B2PIN) == 0);
  }

}

Accès direct au ports d'E/S
Les E/S de l'Arduino sont reliées aux ports d'E/S du microcontroleur Atmega. Les PINs  0,1, 2, ...7 sont reliées au port D. Les PINs 8 à 13 sont reliées au port B. Les PINs A0 à A5 sont reliées au port C.
Chaque port est géré par 3 registres. Prenons le cas du port D, comme on peut le voir sur la figure ci-dessous, il est géré par les registres:
DDRD: registre de direction. C'est ce registre qui précise si on veut utiliser le port en entrée ou en sortie. Chaque bit de DDRD configure la direction de la patte correspondante:  1=> sortie, 0=> entrée,
DDRD = B00001111; signifie que les pattes 0, 1, 2, 3 sont des sorties et les pattes 4, 5, 6, 7 sot des entrées
PORTD: Registre de sortie. Si on place un bit de PORTD à 1 ou à 0, la sortie correspondante passe à 5V ou à 0V
PIND: Registre d'entrée. On le lit pour savoir l'état des broches configurées en entrée.

  •   Pour définir d'un coup les 8 bits d'un registre, une simple affectation suffit. Exemples:
DDRB = B11001100;
PORTB = B11110000;
  • Pour lire le contenu d'un port configuré en entrée, il suffit d'affecter son contenu à une variable. On peut aussi l'utiliser directement dans une expression
x = PINB;
...
if (PINB == B11001110) {
...
}

  • Pour forcer un bit à 1, on fait un OU  avec 1,
PORTD |= B00001000;    //  forcer le bit PD3 à 1

  •   Pour forcer un bit à 0, on fait un ET avec  0,
PORTD |= B00001000; //  forcer le bit PD3 à 1
  • Pour inverser un bit, on fait un XOR avec 1, 
PORTD ^= B00000001; //  forcer le bit PD0 à 1

Interface de communication I2C

L'Arduino UNO possède un module de communication I2C pour communiquer avec tout autre composant disposant du même standard.


Bref rappel du standard I2C

Le protocole I2C (Inter-Integrated Circuit) est un standard de communication synchrone adapté à la communication entre circuits électroniques. La partie Hardware est constituée d'un bus de deux fils SCL (Serial Clock) et SDA (Serial Data). Les deux fils doivent être reliés à 5V par des résistances de pull-Up.


On peut relier plusieurs circuits à un bus I2C, mais pendant une communication, un des circuits est le Master, c'est lui qui génère l'horloge SCL et initie les séquences de communication, les autres sont des slaves.
Les slaves peuvent,  à la demande du master, recevoir et émettre des données sur la ligne SDA. Dans un sens, ou dans l'autre, les bits sont toujours placés sur la ligne SDA au rythme de l'horloge SCL générée par le master.
Un bus peut être connecté à plusieurs masters, mais dès qu'un master prend le contrôle du bus, les autres doivent attendre jusqu'à ce que le bus se libère.
Chaque Slave a une adresse, au début d'une séquence de communication, le Master qui initie la séquence envoie l'adresse du slave avec lequel il désire communiquer, celui-ci reconnaît son adresse et répond, les autres slaves (s'il y en a) restent muets.

Bus Libre

Quand le bus est libre, les deux lignes SDA et SCL sont au niveau haut

Start condition

Quand un Master veut prendre le contrôle du bus et démarrer une séquence de communication, il génère un signal Start condition sur le bus.


  stoP condition

Quand un Master veut terminer une séquence de communication et libérer le bus, il génère un signal stoP condition.

  Écrire un bit sur le bus



L' acknowledge

L'acknowledge est l'accusé de réception. Il est placé par le circuit (qui vient de recevoir un octet) sur la ligne SDA pendant le 9ème coup d'horloge.
•    Si c'est le master qui envoie, alors c'est le slave qui place l'acknowledge
•    Si c'est le master qui reçoit, alors c'est lui même qui place l'acknowledge

o    SDA=0 ==> acknowledge positif (ACK),
o    SDA=1 ==> acknowledge négatif (NoACK),

Début d'une séquence de communication

Une séquence de communication doit toujours commencer comme suit:
•    Le master envoie un Start,
•    Le master envoie l'adresse du Slave sur 7 bits
•    Le master envoie le bit R/W pour indiquer s'il veut envoyer des données vers le slave ou s'il veut en recevoir

o    R/W = 0  =>   il veut envoyer
o    R/W = 1  =>   il veut recevoir

•    Le slave place l'acknowledge pendant le 9ème coup d'horloge


Séquence de transmission (master → slave)



Séquence de réception (master  ← slave)


  • Le master envoie le Start, suivi de l'adresse du Slave avec le bit R/W=1 pour préciser qu'il démarre une séquence de réception
  • Le slave envoie ACK pendant le 9ème coup d'horloge suivi d'un octet pendant les 8 coups d'horloge suivant
  • Le master envoie un ACK (positif) pour préciser qu'il désire recevoir un autre octet
  • Le slave envoie un octet
  • Le master envoie un NoACK (négatif) pour préciser qu'il ne veut plus recevoir
  • Le master envoie un stoP pour mettre fin à la communication et libérer le bus

Séquence bidirectionnelle

Si le master veut envoyer des données puis en recevoir d'autres. Au moment où il envoie le stoP à la fin de la séquence de transmission, le bus devient libre et il peut être pris par un autre master. Pour éviter cela, le master peut envoyer un Start avant d'envoyer le stoP, on appelle un RepeatStart


Fréquence de l'horloge

Le standard I2C supporte 5 vitesses de communication,

  • Standard mode (Sm)      ≤ 100 kbit/s,
  • Fast mode (Fm)          ≤ 400 kbit/s,
  • Fast mode plus (Fm+)      ≤ 1 Mbit/s,
  • High-speed mode (Hsm)      ≤ 3,4 Mbit/s,
  • Ultra-fast mode (UFm)     ≤ 5 Mbit/s, unidirectionnel uniquement.

Le capteur de température LM75A

C'est un capteur Numérique. Il a son propre convertisseur analogique numérique. Il communique avec l'extérieur sur un bus I2C


  •  Capteur de température avec un ADC 11 bits [D10 D9 D8 D7 D6 D5 D4 D3 D2 D1 D0] avec une résolution de 0.125 °C (poids de D0)
  •  Plage de fonctionnement -55°C  à +125 °C
  •  Pas très fiable, (possibilité d'erreur) +/- 2°C  dans la plage (-25°C à +100°C)
  •  Le circuit possède 3 entrées A2 A1 A0 qui permettent de fixer son adresse sur un bus I2C. On peut donc avoir jusqu'à 8 circuits sur le même bus I2C. Les 4 autres bits sont fixés à 1001 ce qui donne l'adresse 7 bits 1 0 0 1 A2 A1 A0
  • Le circuit contient 4 registres internes [simple(8bits) ou double(16 bits)] que l'on peut écrire ou lire. On peut les adresser à l'aide d'un pointeur dont seulement deux bits sont utilisés.    pointeur = [0 0 0 0 0 0 B1  B0]
B1  B0
0  0  Registre de température (2 octets)
0  1  Registre de configuration (1 octet)
1  0  Registre Hystérésis (1 octet)
1  1  Overtemperature shutdown (1 octet)
  • A la mise sous tension, le circuit démarre en mode MORMAL. Dans ce mode, il réalise une mesure de température toutes les 100ms et stocke le résultat dans le registre de température


Le poids du bit de de plus faible poids est 0.125 ==>
- Le poids du MSB = 1
- Le poids du LSB (après cadrage à droite) est 0.125
- La température est alors obtenue par T = MSB + (LSB>>5)*0.125



Ecrire dans un registre interne

  • On envoie le Start suivi de l'adresse I2C du circuit avec le bit R/W=0 pour préciser que c'est une séquence d'écriture. le LM75A retourne un ACK pendant le 9ème coup d'horloge
  • On envoie le pointeur avec l'adresse du registre ciblé. le LM75A retourne un ACK pendant le 9ème coup d'horloge
  • On envoie la donnée à écrire dans le registre ciblé. le LM75A retourne un ACK pendant le 9ème coup d'horloge.
  • On envoie un STOP pour terminer l'échange


Lecture d'un registre interne

Pour lire un registre interne, on a deux possibilités:

  • Soit on précise le pointeur pour indiquer le registre ciblé
  • Soit on ne précise pas le pointeur. Dans ce cas, la valeur actuelle du pointeur est utilisée. Cette valeur a été fixée lors d'un échange précédent. A la mise sou tension, la valeur du pointeur est 0.


Pour lire le registre de température:

  • On envoie le Start suivi de l'adresse I2C du circuit avec le bit R/W=0 pour préciser que derrière on va envoyer le pointeur (adresse interne). Le LM75A retourne un ACK pendant le 9ème coup d'horloge
  • On envoie le pointeur avec l'adresse 00 du registre de température. le LM75A retourne un ACK pendant le 9ème coup d'horloge
  • On envoie un RepeatStart (pour changer le sens de communication sans perdre le contrôle du bus)
  • On envoie l'adresse I2C du circuit avec le bit R/W=1 pour démarrer une lecture. Le LM75A retourne un ACK pendant le 9ème coup d'horloge
  • Le LM75A envoie l'octet MSB (signé). Nous validons par un ACK pendant le 9ème coup d'horloge
  • Le LM75A envoie l'octet LSB (3 bits cadrés à gauche). Nous validons par un N_ACK pendant le 9ème coup d'horloge pour préciser qu'on a fini
  • On envoie un STOP

Le capteur de température et d'humidité SHT31

  • Plage de fonctionnement température:     -40 °C  à +125 °C
  • Plage de fonctionnement humidité:      0%  à 100%
  • Résolution Température: 0.01 °C
  • Résolution humidité: 0.01 %
  • Fiabilité température:   ±    0.2°C  dans la plage [0, 90]°C
  • Fiabilité humidité:  ±   2%
  • Le circuit possède  une seule entrée A qui permettent de fixer son adresse sur un bus I2C. Cette patte doit être reliée à 0V ou à 5V mais ne doit pas être laissée en l'air. Les 6 autres bits de l'adresse sont fixés à 100010 ce qui donne l'adresse 7 bits 100010A. Les deux adresses possible sont 1000100=0x44 ou 1000101=0x45. On peut donc avoir au maximum 2 circuits SHT3x sur le même bus I2C.
  • Pour prendre une mesure, on envoie une commande (2 octets) et le circuit nous renvoie les mesures (6 octets). Deux modes de fonctionnement sont possibles, le mode single  shot, et le mode périodique, On ne parlera dans ce document que du mode single  shot. 

Voir le datasheet pour plus de détail

Mode single shot

Dans ce mode, on envoie une commande pour demander une mesure. Le circuit prend la mesure et nous renvoie les résultats. Il y a 6 commandes différentes pour réaliser cette tâche (tableau ci-dessous). Nous allons essayer d'y voir plus clair. Il y a deux aspects à prendre en considération, la Repeatability et le Clock stretching.
La Repeatability exprime la régularité ou la dispersion des mesures successives autour d'une valeur moyenne. Le tableau ci-dessous montre que le mode High Repeatability donne l'erreur la plus faible (0.04 °C) mais a le temps de conversion le plus long (jusqu'à 15 ms) ce qui engendre aussi une consommation plus élevée car le circuit consomme un courant plus fort pendant les conversions.
Le Clock stretching est un mécanisme du standard I2C. Rappelons les points suivants:

  • Les deux lignes SCK et SDA fonctionnent en mode collecteur ouvert. Si un circuit place une ligne au niveau 0, c'est l'état dominant les autre circuits ne peuvent rien faire
  •  Quand le master envoie une requête de lecture, c'est le slave qui va envoyer une donnée, mais c'est le master qui génère l'horloge SCK, le slave ne fait que placer les bits sur SDA à chaque coup d'horloge.

Que se passe-t-il si le mater démarre l'horloge alors que le slave n'a pas encore préparé la donnée à transmettre (temps de conversion pas encore terminé par exemple). C'est là que rentre en jeux le mécanisme de Clock stretching, que l'on peut résumer ainsi

  • Coté slave: dès que le slave reçoit un Start suivi de son adresse avec le bit R/W=1, il retourne un NoACK pendant le 9eme coup d'horloge puis FORCE L'HORLOGE SCL à 0. Tant que l'horloge est forcée à 0 le master ne peut rien faire.  Dès que le slave a une donnée à transmettre, il libère la ligne SCK et le master rentre en jeux
  • Coté Master:  Si le master reçoit un NoACK après l'envoi d'un Start suivi d'une adresse avec le bit R/W=1, il commence à tester l'état de la ligne SCK (Clock Stretching). Dès qu'elle passe à un, il continue la suite de l'échange

En conclusion, ne sachant pas si la librairie Wire gère le Clock stretching, nous utiliserons les commandes qui n'activent pas ce mode.Il ne reste à choisir entre 3 commandes:

  • Si on prend des mesures à un rythme lent (≈ la seconde), la commande 0x2400   (mode High Repeatability sans clock stretching) semble bien adaptée, elle donne une dispersion plus faible et le temps de conversion de 15ms n'aura pas d'incidence car beaucoup plus faible que la période d'échantillonnage. Le Clock stretching n'étant pas actif, quand on envoie la commande, on attendra 15 à 20ms avant de commencer la lecture des données.
  • Si on prend des mesures à un rythme élevé, (plusieurs mesures par seconde), la commande 0x2416 est plus adaptée car elle a un temps de conversion plus faible ce qui est plus compatible avec un rythme d'échantillonnage élevé. On aura aussi une consommation moyenne plus faible ce qui peut éviter un échauffement du circuit et une altération de la précision. Le Clock stretching n'étant pas actif, quand on envoie la commande, on attendra 4 à 5ms avant de commencer la lecture des données.

Conversion des données en température (°C) et en Humidité (%)

Quand on envoie une commande (Single shot), le circuit réalise une mesure et retourne 6 Octets (unsigned)  Temp_H,  Temp_L,  SRC, Hum_H, Hum_L, SRC

  • Température: On forme un nombre 16 bits NT on collant les deux octets Temp_H et Temp_L,

  • Humidité: On forme un nombre 16 bits NH on collant les deux octets Hum_H et Hum_L,


Exemple

Nous allons utiliser un module SHT31. L'entrée A est reliée à la masse ce qui donne l'adresse 0x44. On va prendre une mesure toutes les 1/2s et afficher les résultats sur le moniteur série.
Remarque: Les mesures ont été réalisées durant le mois de Janvier dans une ville côtière ce qui explique la valeur élevée de l'humidité


#include <Wire.h>

uint8_t B[6];
uint16_t NT, NH;
float T, H;
void setup()
{
  Serial.begin(9600);  // initialisation du port série pour les besoins d'affichage
  Serial.println("Hello");
  Wire.begin();   // initialiser l'interface I2C
  }

void loop()
{
  Wire.beginTransmission(0x44);
  Wire.write(0x24);  // partie haute de la commande
  Wire.write(0x00); // partie basse de la commande
  Wire.endTransmission();  // on valide la transmission et on fini par un stoP
  delay(20); //temps de conversion
  Wire.requestFrom(0x44, 6);
  Wire.readBytes(B,6);
  NT = B[0];
  NT <<= 8;
  NT |= B[1];
  T = NT;
  T *= 175;
  T /= 0xFFFF;
  T = -45 + T;
  NH = B[3];
  NH <<= 8;
  NH |= B[4];
  H = NH;
  H *= 100;
  H /= 0xFFFF;
  Serial.print("Température  =  ");
  Serial.print(T);
  Serial.println(" °C");
  Serial.print("Humidité     =  ");
  Serial.print(H);
  Serial.println("  %");
  Serial.println();
  delay(500);
}


Commander une charge CC

Si on veut commander une charge qui demande plus 200mA ou une charge nécessitant une alimentation supérieure à 5V. Il est évident qu'on ne peut pas la commander directement par une sortie de l'Arduino. Il faut utiliser une interface de puissance capable de fournir à la charge la tension et le courant dont elle a besoin.
La solution la plus simple consiste à utiliser un transistor de commutation. Celui-ci joue le rôle d'interrupteur. Il faut juste choisir un transistor  capable de conduire le courant demandé par la charge quand il est saturé,

  • Il faut calculer Rb pour que le transistor se sature quand la sortie de l'Arduino est au niveau haut
  • Si le courant de la charge est élevé, il peut s'avérer nécessaire d'utiliser un transistor de puissance. Les transistors de puissance ont un 𝛃 assez faible, on risque de se retrouver avec un courant Ib important que le Arduino doit fournir. Pour y remédier, on peut utiliser un transistor Darlington qui a un fort β.



Commander une charge inductive

Dans le cas d'une charge inductive comme un relais ou un moteur, il est IMPÉRATIF de rajouter une DIODE de récupération (roue libre) sinon le transistor sera détruit à la première commutation.
Pour le calcul de Rb, on a besoin de la résistance RL (Continue) de la bobine, il suffit de la mesurer avec un Ohmmètre. Ne vous étonnez pas si vous trouver une valeur faible. C'est normal, c'est juste un fil de cuivre bobiné.






Commande d'une charge alimentée par secteur

Plusieurs solutions sont possibles. Nous exposons ici la solution avec un relais. Elle présente l'avantage d'une isolation galvanique et la possibilité de contrôler des charges de forte puissance.
Le transistor joue le rôle d'interrupteur pour alimenter ou non la bobine du relais. Ainsi le courant qui alimente la bobine n'est pas fourni directement par la sortie de l'Arduino mais il vient de l'alimentation du relais Vrel. Cette dernière peut être différente de celle de l'Arduino dans le cas où le relais nécessite une alimentation > 5V
La résistance Rb doit être calculée pour que le transistor soit saturé quand la sortie de l'Arduino est au niveau haut.


Exemple:
Rrel = 43Ω,  β=100,  Vrel = 5V, Vcc = 5V
Rb  doit être inférieure à   100 ×  43 Ω = 4.3 k,  On prend Rb=3.3k
Quand la sortie de l'Arduino est au niveau haut, elle fournit un courant Is =(5 – 0.7)/3.3k = 1.3 mA
Ce courant sature le transistor qui devient un circuit fermé. La bobine est alors traversée par un courant Ic ≈  5V / 43Ω = 116mA qui vient directement de l'alimentation Vrel. Le courant dans la bobine crée un champ magnétique suffisant pour faire coller le relais, la charge est alors alimentée par le secteur 220V

char c=0;
int pwm = 127;
#define ENA 9
#define IN1 2
#define IN2 3

void setup() {
  TCCR1B = TCCR1B & B11111000 | B00000100;  // 500 Hz
  pinMode(ENA, OUTPUT);
  pinMode(IN1, OUTPUT);
  pinMode(IN2, OUTPUT);
  Serial.begin(9600);
  digitalWrite(IN1,LOW);
  digitalWrite(IN2,LOW);
  analogWrite(ENA,0); // roue libre au déppart

}

void loop() {
  if (Serial.available() > 0){
    c = Serial.read();
    switch (c) {
    case 'G':
      digitalWrite(IN1,LOW);
      digitalWrite(IN2,HIGH); // gauche
      analogWrite(ENA,pwm);   // vitesse
      break;
    case 'D':
      digitalWrite(IN1,HIGH);
      digitalWrite(IN2,LOW);
      analogWrite(ENA,pwm);
      break;
    case 'L':
      analogWrite(ENA,0); // roue libre
      break;
    case 'S':
      digitalWrite(IN1,LOW);  // frein
      digitalWrite(IN2,LOW);
      digitalWrite(ENA,HIGH);
      break;
    case '+':
      pwm += 5;
      if (pwm > 255) pwm = 255;
      analogWrite(ENA, pwm);
      break;
    case '-':
      pwm -= 5;
      if (pwm < 0) pwm = 0;
      analogWrite(ENA, pwm);
      break;
    }
  }
}
