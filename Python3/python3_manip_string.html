Manipulation des chaînes de caractères
Une chaîne de caractère est une séquence de caractères. Rappelons que sous python, les séquences représentent une catégorie d'objets importante comme les listes et les tuples.

Qui dit séquence, signifie que chaque élément est accessible par son index. Une chaîne peut être manipulée comme tel (toute entière) mais on peut aussi accéder à un caractère donné ou à une portion de la chaîne à l'aide des index exactement comme on le fait avec les listes.

>>> s1 = 'Chaine'
>>> s2 = ' de caractères'
>>> s = s1 + s2               => 'Chaine de caractères'   ( + est l'opérateur de concaténation)
>>> s[1]          =>            'h'
>>> s[7:9]       =>            'de'
>>> s[-10:]      =>            'caractères'
>>> len(s)       =>            20
>>> s[3]='î'     =>            TypeError: 'str' object does not support item assignment

Comme on peut le constater sur la ligne précédente, on ne peut pas modifier un élément d'une chaîne. Une chaîne de caractères est une séquence non mutable c.à.d. non modifiable.

 

Opérateurs sur chaînes de caractères

  • Affectation ou définition:   =
s1 = 'Bonjour'
s2 = "Bonsoir\n"

  • Concaténation:    +
s1 = 'Bonjour'
s2 = " cher ami"
s3 = s1 + s2    =>  'Bonjour cher ami'

  • Répétition:    *
s = 5 * 'Blabla'               =>   'Blabla Blabla Blabla Blabla Blabla '
  • Test d'égalité:   ==
s1 == s2          => retourne True si s1 est la même chaine que s2

if   s  ==   'Error':
      print("Il s'est produit une erreur")

  • Test d'appartenance:   in
s1 in s2           => retourne True si s1 est in incluse dans s2
s = 'Bonjour cher ami'
'cher' in s        => retourne True
'Cher' in s       => retourne False

  • Extraction:  [ ]
Une chaine de caractère est une séquence de caractères. On va retrouver les mêmes mécanisme qu'on utilise avec les listes et les tuples. Une chaine peut être manipulée comme tel (toute entière) mais on peut aussi accéder à un caractère donné ou à une portion de la chaine à l'aide des indices (indexes) exactement comme on le fait avec les listes.
s1 = 'Chaine'
s2 = ' de caractères'
s = s1 + s2      => 'Chaine de caractères'
s[0]                 =>            'C'
s[1]                 =>            'h'
s[7:9]              =>            'de'
s[-1]                =>            's'
s[-4]                =>            'è'
s[-10:]             =>            'caractères'
 s[3] = 'î'         =>            TypeError: 'str' object does not support item assignment

Sur la dernière ligne, on a essayé de modifier un caractère de la chaine en lui affectant une nouvelle valeur. On a obtenu une erreur. On ne peut pas modifier un élément d'une chaine. D'où la constation fondamentale: Une chaine de caractère est une séquence non modifiable.

 

Les fonctions s'appliquant sur les chaînes de caractères

  • len(s)            => retourne un entier = longueur de la chaîne
  • chr(n)          => retourne le caractère correspondant à l'identifiant Unicode n
  • ord(c)           => Retourne l'identifiant Unicode du caractère c
  • bytes(s, code)    =>  retourne une séquence d'octets (non mutable):
    b = bytes('ABéçCD', 'utf-8')  --> b'AB\xc3\xa9\xc3\xa7CD'
    l = list(b)   -->  [65, 66, 195, 169, 195, 167, 67, 68]
  • bytearray(s, code)   => retourne une séquence d'octets (mutable)

 

Les méthodes de la classe chaîne de caractère

Voici quelques méthodes s'appliquant sur les chaînes de caractères, d'autres sont énumérée ici: http://docs.python.org/3.3/library/stdtypes.html#string-methods

 

  str.capitalize()
 		Retourne une copie de str avec la première lettre en majuscule
str.count(s, [start],[ stop])
 		Retourne le nombre d'occurrence de la sous-chaine s dans la portion de la chaine str qui commence à start et se termine à stop
 cc = 'ABC ABCDE BCD CDEF ABCDEF'
print(cc.count('BC'))      => 4     =>  on compte dans toute la chaine cc
print(cc.count('BC',3))   => 3      =>   on compte à parir de la position 3
print(cc.count('BC',3,14))   =>   2  =>  de la position 3 jusqu'à la position 14 (non incluse)

str.encode([code],[ erreur])
Retourne une séquence d'octets de type bytes correspondant au codage de la chaine str dans la norme code.
Par défaut, code = 'utf-8', il peut prendre 'ascii', 'latin-1',' iso8859_6', . . .
Par défaut, erreur= 'strict',  elle peut prendre 'ignore', 'replace', 'xmlcharrefreplace', 'backslashreplace'.

Pour plus de détail, voir: http://docs.python.org/3.3/library/codecs.html
 cc = 'leçon'
 print(cc.encode())  =>  b'le\xc3\xa7on' => en utf-8, 'ç' est codé sur deux octets: 0xC3 et 0xA7
 print(cc.encode('latin-1'))  =>  b'le\xe7on'  => 'ç' est codé sur un octet: 0xE7
 print(cc.encode('ascii'))  => erreur 'ascii' codec can't encode character '\xe7'   not in range(128)
str.endswith(s, [start],[ stop])
 		Retourne True si str[start:stop] se termine par la chaine s
print('ABCDE'.endswith('E'))  =>  True
str.find(sub, [start],[ stop])
 		Retourne l'indice de la première occurrence de la chaine sub dans str[start:stop]. Retourne -1 si la chaine sub n'est pas trouvée. voir aussi .rfind
  str.index(sub, [start],[ stop])
 		Similaire à find mais retourne une erreur si la chaine sub n'est pas trouvée
str.isalnum()
 Retourne True si tous les caractères de str sont alphanumériques et qu'il ya au moins un caractère.
  str.isalpha()
 		Retourne True si tous les caractères de str sont alphabétiques et qu'il y a au moins un caractère.
'Bonjour'.isalpha()   =>   True
' Napoléon 3'.isalpha()   =>   False

 str.isdecimal()
Retourne True si tous les caractères de str sont des chiffres décimaux et qu'il y a au moins un chiffre.
'34568'.isdecimal()      =>   True
'345.68'.isdecimal()  =>   False

 str.islower()
Retourne True si tous les caractères de str sont minuscules et qu'il ya au moins un caractère

'abcd'.islower()  =>    True
'Abcd'.islower()  =>    False
'abcd34'.islower()  =>    True
'346'.islower()  =>    False
  str.isspace()
 		Retourne True si str n'est constituée que d'espaces
  str.upper()
 		Retourne True si tous les caractères de str sont majuscules et qu'il ya au moins un caractère
'ABCD'.isupper()  =>    True
 'ABCd'.isupper()  =>    False
 'ABCD34'.isupper()  =>    True
 '346'.isupper()  =>    False

str.join()
 		colle des chaines en utilisant un séparateur
sep = '-'
s = ['ABC', 'XYZ', '123']
 cc = sep.join(s)  =>   'ABC-XYZ-123'
  str.ljust(N,[c])
 		complète une chaine par un caractère en la justifiant à gauche. Voir rjust
'ABC'.ljust(10)  =>  'ABC       '
'ABC'.ljust(10,'.')  => 'ABC.......'
  str.lower()
 		retourne une copie de str mais avec tous les caractères minuscules
'AbbdDDExyz'.lower()   =>        'abbdddexyz'
str.lstrip(['caractèrres'])
 		supprime les caractères indésirable du coté gauche. On arrête de supprimer dès le premier caractère ne faisant pas partie de la liste indésirable.
'       ABCD EFG  HIJ       '.lstrip()  =>   'ABCD EFG  HIJ       '
 'acbghkllw'.lstrip('axbyc')     =>   'ghkllw'
  str.partition(sep)
 		Coupe une chaine à la première occurrence d'un séparateur. Retourne un tuple constitué de la chaine avant le séparateur, du séparateur et de la chaine après le séparateur
'ABCD-EFGH-XYZ'.partition('-')   =>     ('ABCD', '-', 'EFGH-XYZ')
str.replace(old, new, [N])
 		Retourne une copie de la chaine str où toute occurrence de la sous-chaine old est remplacée par la sous-chaine new. Si N est précisé, seuls N occurrences sont remplacées.
cc = 'ABCD\rEFG'.replace('\r','\n')
str.rfind(sub, [start],[ stop])
 		Retourne l'indice de la dernière occurrence de la chaine sub dans str[start:stop]. Retourne -1 si la chaine sub n'est pas trouvée. (voir aussi .find)
'ABCAXYADFG'.rfind('A')   =>     6
  str.rindex(sub, [start],[ stop])
 		Similaire à rfind mais retourne une erreur si la chaine sub n'est pas trouvée
  str.rjust(N,[c])
 		complète une chaine par un caractère en la justifiant à droite. Voir ljust
'ABC'.ljust(10)  =>  '       ABC'
'ABC'.ljust(10,'.')
 '.......ABC'
  str.rpartition(sep)
 		Coupe une chaine à la dernière occurrence d'un séparateur. Retourne un tuple constitué de la chaine avant le séparateur, du séparateur et de la chaine après le séparateur
'ABCD-EFGH-XYZ'.rpartition('-')   =>     ('ABCD-EFGH', '-', 'XYZ')
  str.split([sep],[n])
 		Coupe une chaine en plusieurs sous chaines. Si n est précisé, seuls n sous chaines sont extraites à partir de la gauche.
'ABC DEF GHI IJK'.split()  =>    ['ABC', 'DEF', 'GHI', 'IJK']
 'ABC DEF GHI IJK LMN OPQ'.split(' ',2)   =>    ['ABC', 'DEF', 'GHI IJK LMN OPQ']
  str.rsplit([sep],[n])
 		Coupe une chaine en plusieurs sous chaines. Si n est précisé, seuls n sous chaines sont extraites à partir de la droite.
'ABC DEF GHI IJK'.split()  =>    ['ABC', 'DEF', 'GHI', 'IJK']
 'ABC DEF GHI IJK LMN OPQ'.rsplit(' ',2)   =>   ['ABC DEF GHI IJK', 'LMN', 'OPQ']
str.rstrip(['indésirables'])
 		Supprime les caractères indésirable du coté droite. On arrête de supprimer dès le premier caractère ne faisant pas partie de la liste indésirable.
'       ABCD EFG  HIJ       '.lstrip()  =>   '       ABCD EFG  HIJ'
 'ghkllwacb'.lstrip('axbyc')     =>   'ghkllw'

 str.splitlines(keepends)
 		Casse la chaine str en plusieurs lignes et les place dans une liste
'ABC\rDEF\nHIJ\r\nXYZ\n'.splitlines(False)  =>  ['ABC', 'DEF', 'HIJ', 'XYZ']
 'ABC\rDEF\nHIJ\r\nXYZ\n'.splitlines(True)   =>  ['ABC\r', 'DEF\n', 'HIJ\r\n', 'XYZ\n']
str.startswith(s, [start],[ stop])
 		Retourne True si str[start:stop] commence par la chaine s
print('ABCDE'.endswith('E'))  =>  True
  str.strip(['indésirables'])
 		Supprime les caractères indésirable de droite et de gauche. On arrête de supprimer dès le premier caractère ne faisant pas partie de la liste indésirable.

 'ababghkllwacb'.strip('axbyc')     =>   'ghkllw'
  str.swapcase()
 		Retourne une copie de str en remplaçant les minuscules par des majuscules et vice versa
  str.title()
 		retourne une copie de str avec la première lettre en majuscule et les autres en minuscule.
  str.upper()
 		retourne une copie de str mais avec tous les caractères majuscule
'AbbdDDExyz'.lower()   =>        'ABBDDDEXYZ'


Encodage des caractères

Sous python 3, chaque caractère d'une chaine est codé en Unicode. Unicode affecte un identifiant entier unique pour chaque caractère mais ne précise pas sur combien d'octets faut-il coder cet identifiant. Par exemple, l'identifiant Unicode de ω est 0x3C9 soit 969. Unicode laisse le chois aux développeurs de langages de programmation coder cet identifiant sur le nombre d'octet qui leur convient et ceci en fonction de la plateforme sur laquelle ils travaillent.  Ainsi une chaine peut être constituée de caractère appartenant à plusieurs langues différentes 'ABcéXçαβپيطون'
Tant qu'on travaille sur un système, cela ne pose aucun problème. Mais dès qu'il s'agit d'échanger des données entre systèmes, il faut que les deux systèmes qui échangent se mettent d'accord sur la façon dont les identifiants Unicode sont représentés. Si on envoie des données à une imprimante sans lui préciser comment ces donnés ont été codées, elle risque d'afficher n'importe quoi.
Les échanges entre systèmes se font toujours sous forme de séquences d'octets. C'est pour cette raison que python dispose d'un type appelé byte qui est une suite d'octets avec les mécanismes nécessaires pour passer de la représentation string qui est une suite de caractères à la représentation byte qui est une suite d'octets, et ceci dans les deux sens. (voir les méthodes .encode() et .decode() et les fonctions bytes() et bytearray())
Mais avant de pouvoir passer d'une représentation à une autre il faut définir le nombre d'octet nécessaire pour coder un seul caractère. C'est là qu'interviennent les normes de codages:

  • ASCII, chaque caractère est codé sur 7 bits. On peut coder 128 caractères différents. Il suffit pour écrire un texte 'normal' en anglais. Les 32 premier caractères sont des caractères de control, ils ne sont pas affichables.
  • Latin-1 connu sous Windows comme CP1252 ou plus généralement comme ISO-8859-1. C'est un des tableaux de la norme ISO-8859. Cette norme est une extension de du code ASCII. Chaque caractère est codé sur 8 bits. On peut ainsi coder 256 caractères différents en gardant la compatibilité avec le code ascii. Les caractères de code 0 à 127 sont les mêmes que le code ASCII. Les codes de 128 à 255 sont réservés aux caractères nationaux. Latin-1 permet de coder les caractères accentués des pays de l'Europe de l'Ouest

On peut trouver quelques tableaux de codes de la norme ISO-8856 dans un vieux document que j'ai publié il y a assez longtemps:   http://www.oumnad.123.fr/Archi-Ordinateurs/STO.pdf

  • Utf-8 est une norme récente fortement utilisée. Elle maintient la compatibilité avec le code ASCII standard, les 128 premiers caractères sont codés sur un octet et ils ont gardé leur code ascii. les autres sont codés sur 2, 3 ou même 4 octets. Le tableau ci-dessous donne un aperçu, (unicode, caractère, utf-8). On ne peut pas représenter la totalité du tableau utf-8.


Exemples:
'ABCDE'        ASCII --> un octet (7 bits) par caractère    ==>  65  66  67  68  69
'ABCDE'        Latin-1 --> un octet (8 bits) par caractère    ==> 65  66  67  68  69
'ABCéçDE'     Latin-1 --> un octet (8 bits) par caractère    ==> 65  66  67  233  231  68  69
'ABCéçDE'    ASCII --> 7 bits par caractère    ==>  impossible, les identifiants de é et le ç ne peuvent être représentés sur 7 bits
'ABCéçDE'    Utf-8    -->  65  66  67  195  169  195  167  68  69
'ABéDω'        Utf-8    -->  65  66  195  169  68  207  137
'ABéçDω'       Latin-1    -->  impossible, l'identifiant Unicode de ω est \u03c9 = 669, il ne peut être codé sur 8bits

Pour réaliser l'encodage,  on peut utiliser la méthode .encode() ou les fonctions bytes() et bytearray().
