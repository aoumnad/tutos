Les types de données
La représentation des donnés numériques ou alphanumérique sous python n'est pas triviale, Nous allons simplifier le mieux Possible.

Sous python, on n'est pas obligé de déclarer les types des variables. Le type d'une variable est déterminé automatiquement quand on lui affecte une valeur. Le type peut changer automatiquement par la suite. Par exemple: si on fait x = 123, la variable sera codée comme un entier (int). Si par la suite on fait x=x/2, la variable x prend la valeur 61.5 et devient automatiquement un réel (float).

La fonction type() permet de savoir quel est le type d'une variable.

>>> n = 5
>>> r = n/2
>>> type(n)     =>            <class 'int'>
>>> type(r)     =>            <class 'float'>

Les entiers

Les entiers sont normalement codés sur 32 bits. Mais quand la taille d'un entier dépasse 32 bits, python le code automatiquement sur un nombre d'octet variable. Le programmeur n'a pas à s'en occuper. Rappelons qu'avec 32 bits, on peut coder 232 entiers différents. La moitié (231) sont des nombre positifs (de 0 à 231 - 1) et l'autre moitié sont des nombres négatifs (-1 à -231)

  • Pour déterminer la valeur du nombre entier le plus grand sur une plate-forme donnée
import sys
M = sys.maxsize  # retourne l'entier de base le plus grand, en général 231-1 = 2147483647
print(M)
  • Pour déterminer le nombre de bits nécessaires pour coder un entier M non signé, on peut utiliser la méthode .bit_length() de la classe int.
M = 255
Print(M.bit_length())  # retourne 8
N = 256
Print(M.bit_length())  # retourne 9
  •  Un nombre peut être présisé dans n'importe quelle base
x = 1234          :  entier positif exprimé en décimal
 x = -321           :  entier négatif exprimé en décimal
x = 0xFF          :   Entier exprimé en Hexadécimal. Même chose que x = 255
 x = 0b10011    :  Entier exprimé en binaire. Equivalent à x = 19

Remarque: Qu'un entier doit définit en décimal, en Hexadécimal ou en binaire, à l'intérieur de la machine, il est toujours codé en binaire. Si on l'affiche avec une simple instruction du genre print(x), il sera affiché en décimal. Pour l'afficher dans une autre base, il faut utiliser une sortie formatée.

  •  Pour convertir un entier en une séquence de bytes, on utilise la méthode .to_bytes(length, byteorder, *, signed=False).
    N = 65
    Nb = N.to_bytes(1, 'big')   -->  b'A'
    Nb[0]   -->  65

    N = 250000
    bin(N)  -->  '0b11 11010000 10010000'
    Nb = N.to_bytes(3, 'big')  -->   b'\x03\xd0\x90'
    Kb = N.to_bytes(3, 'little')  --> b'\x90\xd0\x03'
    Nb[0]  -->  3
    Nb[1]  -->  208
    Nb[2]  -->  144
    list(Nb)  -->  [3, 208, 144]
    tuple(Nb)  -->  (3, 208, 144)


  • Pour former un entier à partir d'une séquence de type bytes, on utilise la classe:
    int.from_bytes(bytes, byteorder, *, signed=False)

    Bs = bytes([3, 4, 5])    -->    b'\x03\x04\x05'
    N = int.from_bytes(Bs, 'big')    -->   197637
    bin(N)    -->   '0b110000010000000101'  -->  11  00000100  00000101

Les réels

Les réels (float) sont représentés en virgule flottante. Ils peuvent représenter les nombres compris (approximativement) entre 5 × 10-324  et 1.79 x 10308  (idem pour les négatif). Un nombre plus grand que 10+308 ne provoque pas de message d'erreur mais il est représenté par inf (infini). Utiliser float('inf') représenter l'infini dans une expression.

x = 3.14
x = 0.001
x = 345.
x = -23.18
x = 12e18     (12 × 1018)
x = 156e-6      (156 × 10-6 )

Les booléens

Une grandeur booléenne ne constitue pas un type proprement dit. C'est un sous-type du type int. Une grandeur booléenne ne  peut prendre que deux valeurs 1 ou 0 ou encore True ou False. En réalité 0 correspond à false et tout ce qui est différent de 0 correspond à true

x = False
y = True
x = not y

Les chaînes de caractères

Une chaîne est constituée de 1 ou plusieurs caractère. Il faut la préciser entre ' ' ou " ". Les doubles guillemets sont utiles quand la chaîne  contient un '

x = 'A'
y = "B"
x = 'Bonjour'
y = "Bonsoir"
z = "Ensemble d'instructions"

Les complexes

Le langage python sait manipuler les nombres complexes
z = 5 + 3j
y = 3.4 + 2.75j
abs(z) retourne le module de z
z.real  retourne la partie réelle de z
z.imag  retourne la partie imaginaire de z
z.conjugate()  retourne le conjugué de z


Le module cmath contient d'autres fonctions de manipulation des complexes. Pour les utiliser il faut importer ce module

import cmath                                    importe le module cmath
cmath.phase(z)                                  retourne l'argument de z en radian
cmath.polar(z)                                   retourne la représentation polaire (module, argument)
cmath.rect(module, argument)        retourne  le complexe correspondant à (module, argument)
cmath.pi                                              retourne la valeur de pi

S on importe le module par: from module import * on n'est pas obligé d'utiliser le nom du module devant chaque fonction. Mais ceci peut entraîner des confusions si une fonction porte le même nom dans deux modules différents

from cmath import *             importe le module cmath
phase(z)                                  retourne l'argument de z en radian
polar(z)                                   retourne la représentation polaire (module, argument)
rect(module, argument)        retourne  le complexe correspondant à (module, argument)
pi                                              retourne la valeur de pi


On n'est pas obligé d'importer un module complet. si je n'ai besoin que de la fonction phase et de la valeur de π

from cmath import phase, pi
a = phase(1+1j)      retourne l'argument de 1+j    en radiant   (0.785 rad)
a = a * 180 / pi       le même argument en degré    (45 °)
b= phase(-1+1j)      retourne l'argument de -1+j    en radiant   (2.356 rad)
b = b * 180 / pi       le même argument en degré    (135 °)
