Le langage python 3 et l'Interface Graphique Tkinter
L'interface graphique (GUI) Tkinter
Plusieurs interfaces graphiques peuvent être utilisées avec python, tkinter, wxPython, pyQT, pyGTK. L'interface tkinter (Tk interface) est installée par défaut avec Python. C'est celle-ci que nous allons utiliser dans ce didacticiel.

 La bibliothèque tkinter est une adaptation de la bibliothèque Tk développée à l'origine pour le langage Tcl. Elle contient le module de base Tk et son extension ttk (Tk themed widgets) qui permet d'obtenir des widgets avec une apparence similaire à ceux du système d'exploitation utilisé en faisant appel à la bibliothèque graphique de ce dernier.

Normalement, le module de base Tk est largement suffisant pour monter une interface graphique. Néanmoins, le module ttk introduit quelques nouveaux widgets (comme combobox) et des améliorations appréciables. Comme ttk ne remplace pas complètement le module Tk, il est possible d'utiliser des widgets de l'un et de l'autre dans une même application

 Un élément graphique comme un bouton,  une case à cocher ou une fenêtre de saisie est appelé widget. Un widget est caractérisé par des options  qui sont ses propriétés et des méthodes qui désignent des actions qu'on peut lui appliquer.

Les options sont précisées entre parenthèses au moment de la création du widget. Les méthodes sont invoqués ensuite.

Exemple 1:
from tkinter import *                    # charger la bibliothèque tkinter
fp = Tk()                                         # Créer la fenêtre principale sous le nom fp
# créer le bouton B1. fp désigne le père du bouton, ici c'est la fentre fp. L'option text permet de définir le texte  affiché  sur le bouton
B1 = Button(fp, text = "Bouton 1")
B1.pack()                                       # positionner le bouton dans la fenêtre

B2 = Button(fp, text = "Bouton 2")               # créer le bouton B2 dans la feinter fp
B2.pack()                                       # positionner le bouton dans la fenêtre
fp.mainloop()                                # activer le gestionnaire d'événements qui va  surveiller ce qui se passe dans la fenêtre

On obtient la fenêtre suivante:

Le fait d'avoir utilisé le positionneur pack() sans aucun paramètre, les widgets sont positionnés automatiquement l'un en dessous de l'autre.

Tkinter dispose de trois positionneurs (geometry manager) : pack(), place() et grid(). Chacun utilise une technique différente pour positionner les widgets dans la fenêtre. Nous auront l'occasion d'en parler dans la suite de ce document. pour le moment, on va juste ajouter le paramètre pady au positionneur pack() pour espacer les bouton verticalement (axe y). on replace la ligne  B1.pack() par B1.pack(pady = 5) et on fait de même pour le bouton 2.
